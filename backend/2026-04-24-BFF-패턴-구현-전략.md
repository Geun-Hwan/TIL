### Context
BFF(Backend for Frontend) 패턴은 클라이언트의 특성(웹, 모바일 앱, 데스크톱 등)에 따라 최적화된 API 게이트웨이를 별도로 구성하는 아키텍처 패턴입니다. 범용적인 API(General-purpose API)가 모든 클라이언트의 요구사항을 수용할 때 발생하는 오버페칭(Over-fetching)이나 언더페칭(Under-fetching) 문제를 해결하기 위해 도입합니다.

### Core
Node.js(Express)를 사용하여 웹 클라이언트 전용 BFF를 구현하는 예시입니다. 여러 마이크로서비스(User, Product)에서 데이터를 수집하여 웹 UI에 최적화된 형태로 응답을 재구성합니다.

```javascript
// 웹 클라이언트용 BFF 엔드포인트
app.get('/api/web/dashboard', async (req, res) => {
  try {
    // 여러 서비스(User, Product) 호출
    const [user, products] = await Promise.all([
      fetch('http://user-service/profile').then(r => r.json()),
      fetch('http://product-service/recommended').then(r => r.json())
    ]);

    // 클라이언트 요구사항에 맞게 데이터 재조합 (UI 레이아웃 중심)
    const dashboardData = {
      userName: user.name,
      greeting: `환영합니다, ${user.name}님`,
      recommendedItems: products.slice(0, 3) // 상위 3개만 추출
    };

    res.json(dashboardData);
  } catch (error) {
    res.status(500).send({ message: 'BFF Data Aggregation Failed' });
  }
});
```

### Insight
BFF 패턴은 클라이언트 팀이 직접 해당 층을 관리함으로써 API 변경에 유연하게 대응할 수 있는 강력한 이점이 있습니다. 다만, 서비스 수가 많아질 경우 코드 중복 발생 가능성이 있으므로 공통 로직은 공유 라이브러리(Shared Library)로 분리하는 전략이 필요합니다. 모바일 환경처럼 대역폭 최적화가 필수적인 경우, BFF는 데이터 페이로드(Payload)를 줄이는 데 가장 효과적인 해결책이 됩니다.

### Context
마이크로서비스 아키텍처에서 서비스 간 통신 효율을 높이고, 강한 타입(Strongly-typed) 계약을 통해 개발 생산성을 향상시키기 위해 gRPC(gRPC Remote Procedure Calls)를 도입하려는 상황입니다. 기존 REST/JSON 방식 대비 저지연 통신과 명확한 인터페이스 정의가 필요한 환경에서의 활용 전략을 정리합니다.

### Core
gRPC는 HTTP/2를 기반으로 Protocol Buffers(Protobuf)를 사용하여 직렬화 성능을 최적화합니다.

* **Gradle 설정 (Spring Boot)**
```groovy
plugins {
  id 'com.google.protobuf' version '0.9.4'
  id 'java'
}

dependencies {
  implementation 'io.grpc:grpc-netty:1.62.2'
  implementation 'io.grpc:grpc-protobuf:1.62.2'
  implementation 'io.grpc:grpc-stub:1.62.2'
  implementation 'net.devh:grpc-server-spring-boot-starter:2.15.0.RELEASE'
  implementation 'net.devh:grpc-client-spring-boot-starter:2.15.0.RELEASE'
}

protobuf {
  protoc { artifact = 'com.google.protobuf:protoc:3.24.4' }
  plugins {
    grpc { artifact = 'io.grpc:protoc-gen-grpc-java:1.62.2' }
  }
  generateProtoTasks {
    all().each { task -> task.plugins { grpc {} } }
  }
}
```

* **서비스 인터페이스 정의 (.proto)**
```protobuf
syntax = "proto3";
package order.v1;

service OrderService {
  rpc PlaceOrder (PlaceOrderRequest) returns (PlaceOrderResponse) {}
}

message PlaceOrderRequest {
  string user_id = 1;
  repeated OrderLine lines = 2;
}

message OrderLine {
  string product_id = 1;
  int32 quantity = 2;
}

message PlaceOrderResponse { string order_id = 1; }
```

* **서버 측 서비스 구현**
```java
@GrpcService
public class OrderGrpcService extends OrderServiceGrpc.OrderServiceImplBase {
  @Override
  public void placeOrder(PlaceOrderRequest request, StreamObserver<PlaceOrderResponse> responseObserver) {
    String orderId = UUID.randomUUID().toString();
    PlaceOrderResponse response = PlaceOrderResponse.newBuilder().setOrderId(orderId).build();
    responseObserver.onNext(response);
    responseObserver.onCompleted();
  }
}
```

### Insight
gRPC는 단순한 RPC 호출을 넘어 HTTP/2의 멀티플렉싱과 양방향 스트리밍을 통해 마이크로서비스 간 통신 비용을 획기적으로 줄여줍니다. 특히 Protobuf를 통한 스키마 기반의 개발은 팀 간 계약(Contract)을 명확히 하여 호환성 문제를 사전에 방지합니다. 운영 측면에서는 서비스 디스커버리 및 부하 분산을 위해 Envoy와 같은 L7 프록시를 함께 구성하는 것이 권장되며, 인터셉터(Interceptors)를 활용해 로깅, 트레이싱(Tracing), 인증을 중앙화하여 관리하는 설계가 필수적입니다.

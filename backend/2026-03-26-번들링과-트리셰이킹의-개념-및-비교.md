### Context
웹 개발에서 효율적인 성능 최적화를 위해 모듈 번들러(Module Bundler)의 역할은 매우 중요하다. 특히 브라우저가 해석 가능한 파일로 코드를 변환하고 최적화하는 과정에서 '번들링(Bundling)'과 '트리 셰이킹(Tree Shaking)'은 핵심적인 개념이다. 개발 규모가 커질수록 로딩 속도와 파일 크기를 줄이기 위해 이 두 개념을 명확히 이해하고 활용해야 한다.

### Core
번들링과 트리 셰이킹은 상호 보완적인 최적화 기술이다.

* 번들링(Bundling): 여러 개의 파일, 모듈, 자산(Assets)들을 하나 또는 소수의 파일로 합치는 과정이다.
* 브라우저는 HTTP/1.1 환경에서 많은 수의 파일을 불러올 때 오버헤드가 크기 때문에, 이를 방지하기 위해 번들러가 사용된다.
* 대표 도구: Webpack(기본), Vite(esbuild 기반).

* 트리 셰이킹(Tree Shaking): 번들링 과정에서 사용하지 않는 코드(Dead Code)를 찾아내어 최종 결과물에서 제거하는 기술이다.
* ES Modules(ESM)의 정적 구조(Static Structure)를 활용하여, import/export 구문을 분석해 어떤 코드가 실제로 참조되는지 판단한다.

* 비교 요약
* 목적: 번들링은 파일 결합(성능), 트리 셰이킹은 코드 제거(용량 최적화).
* 의존성: 트리 셰이킹은 번들링 과정의 일부로 작동한다.
* 작동 방식: 번들링은 모든 의존성을 그래프화하며, 트리 셰이킹은 이 그래프에서 참조되지 않는 노드를 '흔들어 떨어뜨린다'.

### Insight
번들링은 현대 웹 개발의 표준이지만, 무분별한 번들링은 오히려 초기 로딩 성능을 저하시킬 수 있다. 따라서 번들링 시 코드 분할(Code Splitting)을 병행하여 필요한 코드만 로드하는 전략이 필요하다. 또한, Vite와 같은 최신 도구는 esbuild를 활용해 번들링 시간을 획기적으로 단축하고, Rollup을 통해 정교한 트리 셰이킹을 수행한다. 효율적인 빌드 환경을 구성하려면 모듈 시스템(ESM)을 준수하는 라이브러리를 선택하는 것이 매우 중요하다.

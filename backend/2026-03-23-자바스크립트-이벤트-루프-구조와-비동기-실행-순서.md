### Context
자바스크립트는 싱글 스레드(Single-threaded) 언어임에도 불구하고, 비동기 처리를 통해 효율적인 작업을 수행합니다. 이 메커니즘을 이해하는 것은 자바스크립트의 실행 흐름을 제어하고 성능을 최적화하는 데 핵심적입니다.

### Core
자바스크립트의 비동기 실행 흐름은 다음과 같은 구성요소와 우선순위 규칙에 따라 동작합니다.

* 실행 우선순위: Call Stack > Microtask Queue > Callback Queue (Task Queue)

```javascript
console.log('1. 동기 코드 실행');

setTimeout(() => {
  console.log('4. Callback Queue (Task Queue)');
}, 0);

Promise.resolve().then(() => {
  console.log('3. Microtask Queue (Promise)');
});

console.log('2. 동기 코드 실행 완료');

// 실행 결과: 1 -> 2 -> 3 -> 4
```

* Call Stack: 현재 호출된 함수들이 쌓이는 스택 구조입니다. 자바스크립트 엔진은 이곳이 비어있을 때까지 작업을 처리합니다.
* Web APIs: setTimeout, fetch와 같은 비동기 작업을 브라우저(또는 Node.js 런타임)에서 처리하고 완료되면 큐로 보냅니다.
* Microtask Queue: Promise.then, process.nextTick(Node.js) 등이 포함되며, 일반 큐보다 높은 우선순위를 가집니다.
* Callback Queue: setTimeout, setInterval 등의 콜백 함수가 대기하는 곳입니다.
* Event Loop: Call Stack이 비어있을 때, Microtask Queue를 먼저 모두 비우고 그 다음 Callback Queue의 작업을 Call Stack으로 이동시킵니다.

### Insight
비동기 처리는 단순한 순서 보장이 아니라, 자바스크립트 엔진의 큐 우선순위 관리 정책입니다. 특히 Promise와 같은 마이크로태스크(Microtask)는 동기 코드가 끝난 직후, 즉 렌더링이나 I/O 작업 이전에 즉시 실행되므로 성능 민감한 로직 설계 시 주의가 필요합니다. 과도한 마이크로태스크는 이벤트 루프를 점유하여 메인 스레드를 차단(Blocking)할 수 있습니다.

### Context
웹 접근성(Web Accessibility)과 웹 표준(Web Standards)은 모든 사용자가 신체적, 기술적 제약 없이 웹 콘텐츠에 접근할 수 있도록 보장하는 핵심 가이드라인이다. W3C(World Wide Web Consortium)가 제정한 웹 콘텐츠 접근성 지침(WCAG)을 준수하는 것은 단순히 법적 의무를 넘어, 사용자 경험(UX) 최적화와 검색 엔진 최적화(SEO)라는 실질적인 비즈니스 이점을 제공한다.

### Core
웹 접근성 및 표준 준수를 위한 주요 체크리스트 항목이다.

* 시맨틱 HTML(Semantic HTML) 활용
  * 페이지의 구조를 올바른 태그(header, nav, main, footer, section, article)로 정의하여 스크린 리더기가 콘텐츠의 의미를 파악하도록 한다.
* 이미지에 대체 텍스트(Alternative Text) 제공
  * 모든 `<img>` 태그에 `alt` 속성을 추가하여 이미지를 볼 수 없는 사용자에게 정보를 전달한다.
* 적절한 색상 대비(Color Contrast) 유지
  * 텍스트와 배경색 간의 명도 대비를 WCAG AA 레벨(4.5:1 이상)로 설정하여 저시력 사용자의 가독성을 확보한다.
* 키보드 내비게이션 지원
  * 마우스 없이 Tab 키만으로 모든 대화형 요소(버튼, 링크, 폼 입력)에 접근하고 조작할 수 있어야 하며, 포커스 표시(Focus Indicator)가 명확해야 한다.
* 폼 레이블(Form Label) 명시
  * 모든 입력 필드(`input`)는 `<label>` 태그와 연결되어 있어야 하며, `aria-label`이나 `aria-labelledby`를 통해 보조 기기에서 레이블을 식별할 수 있게 한다.
* 동적 콘텐츠 접근성(ARIA)
  * 자바스크립트로 변경되는 콘텐츠는 `aria-live` 속성을 사용하여 변경 사항을 보조 기기에 실시간으로 알린다.

### Insight
웹 표준을 준수하는 코드는 구조적으로 깔끔하며 유지보수가 용이하다. 접근성을 고려한 마크업은 스크린 리더와 같은 보조 기기뿐만 아니라, 검색 엔진 봇(Crawler)이 웹사이트를 더 정확하게 이해하도록 도와 SEO 성과를 크게 향상시킨다. 웹 접근성은 선택 사항이 아닌 '사용자를 위한 기본 설계'라는 인식의 전환이 필요하다.

### Context
Supabase에서 카카오(Kakao) 소셜 로그인을 구현하기 위해 카카오 디벨로퍼스(Kakao Developers)와 Supabase 대시보드 간의 API 키 연동 및 리다이렉트 URI(Redirect URI) 설정을 수행하는 과정입니다.

### Core
* **카카오 디벨로퍼스 설정**
  * 카카오 디벨로퍼스 로그인 후 [내 애플리케이션]에서 앱 생성
  * [앱 설정] > [요약 정보]에서 REST API 키 확인
  * [제품 설정] > [카카오 로그인] 메뉴로 이동하여 '활성화 설정'을 'ON'으로 변경
  * [제품 설정] > [카카오 로그인] > [보안]에서 'Client Secret' 코드 생성 및 복사
  * [제품 설정] > [카카오 로그인] > [동의 항목]에서 이메일 등 필요한 정보를 '필수 동의'로 설정
  * [비즈니스] > [비즈 앱 전환] (필요 시 선택)

* **Supabase 설정**
  * [Authentication] > [Providers] > [Kakao] 이동
  * Enable Kakao 설정 후, 확보한 REST API 키와 Client Secret 입력
  * 제공된 Callback URL(Redirect URI) 복사

* **리다이렉트 URI 등록**
  * 다시 카카오 디벨로퍼스 [제품 설정] > [카카오 로그인]으로 이동
  * [Redirect URI] 메뉴에서 Supabase에서 복사한 URL을 등록

### Insight
카카오 로그인을 성공적으로 연동하려면 반드시 카카오 디벨로퍼스 내 '카카오 로그인' 기능을 '활성화(ON)' 상태로 변경해야 합니다. 또한, Supabase의 Callback URL은 카카오가 인증 완료 후 사용자 정보를 전달받을 종착지이므로 정확히 일치해야 합니다. 비즈 앱 전환은 일반적인 개인 개발 환경에서는 선택 사항이나, 서비스 규모에 따라 필요할 수 있습니다.

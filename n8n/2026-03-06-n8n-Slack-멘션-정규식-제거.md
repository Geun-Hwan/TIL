### Context
Slack에서 발생한 이벤트 메시지를 n8n을 통해 처리할 때, 시스템상 멘션된 사용자 정보가 `<@USER_ID>`와 같은 텍스트 형태로 포함됩니다. 이러한 멘션 토큰은 실제 Slack 인터페이스에서는 알림으로 처리되어 사용자에게 보이지 않지만, API 응답값(Raw Text)에는 그대로 노출됩니다. LLM 에이전트에게 컨텍스트를 전달하기 전, 불필요한 데이터인 멘션 토큰을 제거하여 프롬프트 품질을 최적화할 필요가 있습니다.

### Core
n8n의 Expression 기능 내에서 자바스크립트(JavaScript)의 `replace()` 메서드와 정규 표현식(Regular Expression)을 조합하여 처리합니다. Slack 멘션 패턴 `<@U...>`를 찾아 빈 문자열로 치환하는 방식입니다.

```javascript
{{ $json.text.replace(/<@[A-Z0-9]+>/g, '').trim() }}
```

* `$json.text`: 원본 메시지 데이터가 담긴 필드
* `/<@[A-Z0-9]+>/g`: 슬래시 사이에 정규식을 정의
    * `<@`: 시작 패턴
    * `[A-Z0-9]+`: 영문 대문자와 숫자로 구성된 사용자 ID 매칭
    * `>`: 종료 패턴
    * `g`: 전역(Global) 플래그를 사용하여 메시지 내 모든 멘션을 제거
* `.trim()`: 제거 후 남을 수 있는 불필요한 앞뒤 공백 제거

### Insight
이 방식은 Slack의 API 응답 구조가 명확하게 고정되어 있을 때 가장 효율적입니다. 만약 멘션 제거 후 텍스트가 엉키는 현상이 발생한다면, 정규식 뒤에 `.replace(/\s+/g, ' ')`를 추가하여 다중 공백을 단일 공백으로 정규화하는 과정을 병행하는 것이 좋습니다. LLM 프롬프트에 불필요한 ID 토큰을 전달하지 않음으로써 입력 토큰 비용을 절감하고, 모델의 응답 정확도를 높일 수 있습니다.

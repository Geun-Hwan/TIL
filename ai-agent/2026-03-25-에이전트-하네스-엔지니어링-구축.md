### Context
LLM을 활용한 에이전트 시스템을 구축할 때, 단순히 프롬프트를 작성하는 것을 넘어 모델의 성능을 측정하고, 반복 가능한 테스트를 수행하며, 입력값과 출력값을 관리하는 인프라가 필요합니다. 이를 '에이전트 하네스(Agent Harness)' 또는 '평가 하네스(Evaluation Harness)'라고 부릅니다. 하네스 엔지니어링은 에이전트가 복잡한 도구 호출(Tool Calling)이나 추론을 수행할 때 발생할 수 있는 오류를 정형화된 테스트 세트로 검증하여 신뢰성을 확보하는 과정을 의미합니다.

### Core
하네스 엔지니어링의 핵심은 '테스트 케이스 관리'와 '평가 파이프라인'입니다. 아래는 간단한 프롬프트 검증을 위한 하네스 구조 예시입니다.

```python
# 간단한 에이전트 하네스 구조 (Python)
class AgentHarness:
    def __init__(self, agent_function):
        self.agent = agent_function

    def run_test(self, test_case):
        # 도구 호출 및 결과 확인
        result = self.agent(test_case['input'])
        return result == test_case['expected']

# 프롬프트 설정 예시
SYSTEM_PROMPT = """
당신은 데이터베이스 전문가입니다. 
제공된 입력값을 기반으로 안전한 SQL 쿼리를 생성하세요.
반드시 예약어는 대문자를 사용하십시오.
"""

# 테스트 케이스
test_suite = [
    {"input": "사용자 목록 보여줘", "expected": "SELECT * FROM USERS;"},
    {"input": "삭제할 아이디는 100", "expected": "DELETE FROM USERS WHERE ID = 100;"}
]
```

### Insight
하네스 엔지니어링은 단순히 코드를 짜는 것이 아니라, LLM의 비결정론적(Non-deterministic) 특성을 제어하기 위한 필수적인 품질 보증(QA) 프레임워크입니다. 
* 평가(Evaluation) 자동화: LLM의 결과물이 기대를 충족하는지 정량적으로 수치화할 수 있습니다.
* 반복성(Reproducibility): 프롬프트 수정 시 이전 성능과 비교하여 회귀(Regression) 여부를 즉시 파악할 수 있습니다.
* 제 견해로는, 에이전트가 고도화될수록 내부 로직보다 이를 감싸는 하네스의 견고함이 서비스의 안정성을 결정짓는 핵심 요소가 될 것입니다.

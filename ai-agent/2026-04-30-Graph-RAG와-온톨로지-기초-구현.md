### Context
최근 RAG(Retrieval-Augmented Generation) 시스템에서 단순 벡터 검색(Vector Search)의 한계를 극복하기 위해 지식 그래프(Knowledge Graph)를 결합한 Graph RAG 방식이 주목받고 있습니다. 온톨로지(Ontology)를 사용하여 데이터 간의 의미적 관계를 명확히 정의하면, 모델이 복잡한 문맥적 질문에 대해 더 정확한 추론을 수행할 수 있습니다. 여기서는 Python의 `rdflib`를 사용하여 간단한 온톨로지를 정의하고, 이를 통해 정보를 구조화하는 기초 예시를 다룹니다.

### Core
RDF(Resource Description Framework)를 사용하여 '개발자'와 '프로그래밍 언어' 간의 관계를 정의하는 온톨로지 예제입니다.

```python
from rdflib import Graph, URIRef, Literal, Namespace, RDF

# 1. 그래프 생성 및 네임스페이스 정의
g = Graph()
EX = Namespace("http://example.org/")

# 2. 데이터 추가 (Ontology Triples: 주어-술어-목적어)
# 개발자(Alice)는 Python을 사용한다.
alice = URIRef(EX.Alice)
python = URIRef(EX.Python)
uses = URIRef(EX.uses)

g.add((alice, RDF.type, EX.Developer))
g.add((alice, uses, python))

# 3. 데이터 조회
for s, p, o in g:
    print(f"{s} -> {p} -> {o}")
```

### Insight
*   **의미적 구조화**: 단순 텍스트 임베딩과 달리 온톨로지는 개체 간의 명시적인 관계(예: 'uses', 'part_of')를 저장하므로, Graph RAG 구축 시 LLM이 데이터 간의 연결성을 명확히 파악하는 데 유리합니다.
*   **확장성**: `rdflib`는 다양한 포맷(Turtle, XML, JSON-LD)을 지원하여 기존 시스템과의 통합이 용이합니다. 다만, 실무적인 Graph RAG를 구현할 때는 `Neo4j`와 같은 그래프 데이터베이스와 LLM을 연동하는 파이프라인(예: LangChain의 GraphIndex)을 검토하는 것이 좋습니다.
*   **추론 성능**: Graph RAG는 단순히 관련된 정보를 찾는 것을 넘어, 지식 그래프의 경로를 따라 다단계 추론(Multi-hop Reasoning)을 수행할 수 있는 기반이 됩니다.

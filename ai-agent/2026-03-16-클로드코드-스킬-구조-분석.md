### Context
개발 에이전트인 클로드 코드(Claude Code)와 기타 코드베이스(Codebase) 기반 도구들에서 '스킬(Skills)'은 에이전트가 특정 작업을 수행하기 위해 외부 도구나 API를 호출하는 기능을 의미합니다. 클로드 코드는 이러한 기능을 확장 가능한 플러그인 형태로 제공하며, 사용자가 직접 정의하거나 기존 도구를 통합하여 개발 생산성을 높일 수 있습니다.

### Core
클로드 코드에서의 스킬 구현 및 설정은 주로 다음과 같은 방식으로 작동합니다.

*   스킬 정의: 클로드 코드는 시스템 프롬프트와 연동하여 사용 가능한 도구(Tools) 목록을 관리하며, 이를 통해 파일 시스템 접근, 터미널 명령어 실행, 코드 분석 등을 수행합니다.
*   사용자 정의 스킬(Custom Skills): 특정 프로젝트에 맞춘 스크립트나 명령어를 클로드 코드가 호출할 수 있도록 설정 파일 내에 등록하여 에이전트의 기능을 확장합니다.
*   구조화된 호출: 클로드 코드는 도구 호출(Tool Calling) 메커니즘을 사용하여 명확한 매개변수와 함께 스킬을 실행하며, 실행 결과를 피드백으로 받아 다음 작업을 결정합니다.

```bash
# 클로드 코드 실행 시 스킬을 확장하거나 컨텍스트를 제공하는 방식의 예시
# 실제로는 .claude/config 또는 환경 변수를 통해 도구 권한을 제어함
claude --config ./my-project-tools.yaml
```

### Insight
현재 클로드 코드는 사용자 정의 스킬을 통해 높은 유연성을 보여주고 있으나, 일반적인 '코덱스(Codebase)' 환경(범용적인 IDE 도구 포함)에서는 스킬을 직접 생성하는 프레임워크가 표준화되어 있지 않은 경우가 많습니다. 클로드 코드는 LLM 기반 에이전트의 특성을 살려 도구 호출(Tool Calling) 기반의 확장성을 극대화하고 있으며, 이는 단순한 자동 완성을 넘어 복잡한 프로젝트 작업을 수행하는 핵심 동력이 됩니다. 타 도구들도 점차 에이전트 기반의 도구 호출 체계를 도입하고 있으나, 클로드 코드만큼 심리스(Seamless)하게 통합된 환경을 구축하는 것이 기술적 차별점입니다.
